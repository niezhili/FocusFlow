# FocusFlow 运行逻辑详解

---

## 一、文件级别运行逻辑

```
┌─────────────────────────────────────────────────────────────────┐
│                       用户操作（点击/滑动）                        │
└──────────────────────────┬──────────────────────────────────────┘
                           ↓
┌─────────────────────────────────────────────────────────────────┐
│  Compose UI Layer (Kotlin)                                      │
│  app/.../focusflow/ui/*/***Screen.kt                            │
│                                                                 │
│  职责：只负责界面绘制和事件转发                                  │
│  事件：onClick → 调用 ViewModel.方法()                           │
│  数据：collectAsState() 观察 ViewModel 的 LiveData              │
└──────────────────────────┬──────────────────────────────────────┘
                           ↓
┌─────────────────────────────────────────────────────────────────┐
│  ViewModel Layer (Java)                                         │
│  app/.../focusflow/ui/*/*ViewModel.java                         │
│                                                                 │
│  职责：业务逻辑 + 状态管理                                       │
│  - TimerViewModel：三态状态机 (IDLE→RUNNING→STOPPED)            │
│  - TaskListViewModel：任务增删改查 + 搜索                        │
│  - StatsViewModel：6项数据并行加载 + 统计计算                    │
│  - SettingsViewModel：偏好读写                                   │
└──────────────────────────┬──────────────────────────────────────┘
                           ↓
┌─────────────────────────────────────────────────────────────────┐
│  Repository Layer (Java)                                        │
│  app/.../focusflow/data/repository/*Repository.java             │
│                                                                 │
│  职责：数据源编排（Room + DataStore）                            │
│  - FocusRepository：session 写入/查询/安全网                     │
│  - TaskRepository：task 写入/查询                                │
└──────────────────────────┬──────────────────────────────────────┘
                           ↓
┌─────────────────────────────────────────────────────────────────┐
│  DAO Layer (Java Interface)                                     │
│  app/.../focusflow/data/dao/*Dao.java                           │
│                                                                 │
│  职责：SQL 操作（Room 编译时自动生成实现）                       │
│  - FocusSessionDao：专注记录表的 CRUD + 聚合统计                │
│  - TaskDao：任务表的 CRUD                                       │
└──────────────────────────┬──────────────────────────────────────┘
                           ↓
                    SQLite 数据库 (focusflow.db)
```

**两条特殊路径：**

```
┌─────────────────────────────────────────────────────────────────┐
│  TimerService（前台服务）                                         │
│  app/.../focusflow/service/TimerService.java                    │
│                                                                 │
│  独立于 UI 进程运行，即使 App 被杀死也不影响计时                  │
│  - onStartCommand() 接收 4 种 Action：START/STOP/PAUSE/RESUME   │
│  - 通知栏显示 [暂停] [停止] 按钮                                 │
│  - onDestroy() 安全网：若 isRunning == true 则自动保存 session   │
└─────────────────────────────────────────────────────────────────┘

┌─────────────────────────────────────────────────────────────────┐
│  DataStore（偏好存储）                                            │
│  app/.../focusflow/data/preferences/UserPreferences.java        │
│                                                                 │
│  独立于 Room 的存储，存主题/提示音/震动等设置                     │
│  Flow 变化 → Compose 自动重组，无需手动刷新                      │
└─────────────────────────────────────────────────────────────────┘
```

## 二、函数级调用链

### 2.1 开始计时全链路

```
TimerScreen.kt
  └─ onStartClick() → viewModel.toggleTimer()
       │
       ▼
TimerViewModel.java:toggleTimer()
  └─ IDLE/STOPPED → startTimer(), RUNNING → stopTimer()
       │
       ▼
TimerViewModel.java:startTimer()
  ├─ 第1步：focusRepository.createSession(taskId, startTime)
  │    │
  │    ▼
  │  FocusRepository.java:createSession()
  │    └─ focusSessionDao.insertSession(new FocusSession(taskId, startTime))
  │         │
  │         ▼
  │       FocusSessionDao.java (Room 自动生成 SQL)
  │         └─ INSERT INTO focus_sessions (task_id, start_time) VALUES (?, ?)
  │
  ├─ 第2步：启动前台服务
  │    Intent("START") → startForegroundService() → TimerService.onStartCommand()
  │    │
  │    ▼
  │  TimerService.java:onStartCommand() → case "START"
  │    ├─ 获取 WakeLock (防止 CPU 休眠)
  │    ├─ startForeground() (通知栏常驻)
  │    ├─ 播放提示音 + 震动
  │    └─ Observable.interval(1s) 秒级定时器
  │
  ├─ 第3步：Observable.interval(1s) 每秒更新 UI
  │    └─ elapsedSeconds++ → updateNotification(timeStr)
  │
  └─ 第4步：timerState.setValue(RUNNING)
```

### 2.2 停止计时全链路（含双路径安全网）

```
TimerScreen.kt
  └─ onStopClick() → viewModel.toggleTimer()
       │
       ▼
TimerViewModel.java:stopTimer()
  ├─ disposables.clear()  // 停止 Observable.interval
  │
  ├─ [路径A：正常路径] executeCompleteSession(currentSessionId)
  │    │
  │    ▼
  │  FocusSessionDao.java:updateSession(session)
  │    └─ Room: UPDATE focus_sessions SET actual_duration=?, end_time=?, is_completed=1 WHERE id=?
  │
  ├─ TaskRepository.addTaskDuration(taskId, elapsedSeconds, now)
  │    │
  │    ▼
  │  TaskDao.java:addFocusSeconds()
  │    └─ Room: UPDATE tasks SET total_focus_seconds = total_focus_seconds + ? WHERE id = ?
  │
  └─ stopForegroundService() → Intent("STOP") → TimerService.onStartCommand()
       │
       ▼
     TimerService.java:onStartCommand() → case "STOP"
       │
       ▼
     TimerService.java:stopTimerAndSave()
       ├─ releaseWakeLock()
       ├─ stopForeground(STOP_FOREGROUND_REMOVE)
       │
       ├─ [路径B：安全网路径，不依赖 session ID]
       │    focusSessionDao.completeSessionByTaskAndStartTime(taskId, startTime, elapsed, now)
       │    │
       │    ▼
       │  FocusSessionDao.java (SQL 安全网)
       │    └─ UPDATE focus_sessions
       │       SET actual_duration=?, end_time=?, is_completed=1
       │       WHERE task_id=? AND start_time=? AND is_completed=0
       │       ← 关键：只更新未完成的记录，防止重复写入
       │
       └─ stopSelf()
       │
       ▼
     TimerService.java:onDestroy()
       ├─ [路径C：最终安全网，系统杀死 Service 时触发]
       │    if (isRunning) {
       │        focusSessionDao.completeSessionByTaskAndStartTime(...)
       │    }
       └─ disposables.clear()
       └─ releaseWakeLock()
```

### 2.3 暂停/继续全链路（通知栏交互）

```
用户点击通知栏"暂停"按钮
  │
  ▼
PendingIntent → Intent(action="PAUSE") → TimerService.onStartCommand()
  │
  ▼
TimerService.java:onStartCommand() → case "PAUSE"
  │
  ▼
TimerService.java:pauseTimer()
  ├─ isRunning = false, isPaused = true
  ├─ disposables.clear()    // 停止秒级定时器
  └─ buildNotification(timeStr + " (已暂停)", isPaused=true)
       └─ 通知更新：按钮变为"继续"

───────────────────────────────────────────

用户点击通知栏"继续"按钮
  │
  ▼
PendingIntent → Intent(action="RESUME") → TimerService.onStartCommand()
  │
  ▼
TimerService.java:onStartCommand() → case "RESUME"
  │
  ▼
TimerService.java:resumeTimer()
  ├─ isRunning = true, isPaused = false
  ├─ disposables.clear()
  ├─ buildNotification(timeStr, isPaused=false)  // 更新通知为运行状态
  └─ Observable.interval(1s)  // 重新启动秒级定时器
```

### 2.4 统计页加载全链路

```
进入统计页 → StatsViewModel 构造方法
  │
  ├─ loadTodayStats()
  │    └─ focusSessionDao.getTodayTotalFocusSeconds(dayStart, dayEnd)
  │         └─ SQL: SELECT COALESCE(SUM(actual_duration), 0) FROM focus_sessions
  │                  WHERE end_time >= ? AND end_time < ? AND is_completed = 1
  │
  ├─ loadTodaySessions()
  │    └─ focusSessionDao.getDaySessions(dayStart, dayEnd)
  │         └─ SQL: SELECT t.title AS date, SUM(f.actual_duration) AS total
  │                  FROM focus_sessions f
  │                  INNER JOIN tasks t ON f.task_id = t.id
  │                  WHERE f.end_time >= ? AND f.end_time < ? AND f.is_completed = 1
  │                  GROUP BY t.id, t.title
  │
  ├─ loadTodayCompletedTasks()
  │    └─ focusSessionDao.getTodayCompletedTaskCount(dayStart, dayEnd)
  │         └─ SQL: SELECT COUNT(DISTINCT task_id) FROM focus_sessions
  │                  WHERE end_time >= ? AND end_time < ? AND is_completed = 1
  │
  ├─ loadConsecutiveDays()
  │    └─ focusSessionDao.getFocusDates()
  │         └─ SQL: SELECT DISTINCT strftime('%Y-%m-%d', end_time/1000, 'unixepoch')
  │                  FROM focus_sessions WHERE is_completed = 1 ORDER BY date DESC
  │    └─ 算法：从今天往前推，遇到没有记录的日期停止计数
  │
  ├─ loadTaskDistribution()
  │    └─ focusSessionDao.getTaskDistribution()
  │         └─ SQL: SELECT t.id, t.title, COALESCE(SUM(f.actual_duration), 0)
  │                  FROM tasks t LEFT JOIN focus_sessions f ON t.id = f.task_id
  │                  GROUP BY t.id, t.title
  │
  └─ loadWeekStats()
       └─ focusSessionDao.getWeeklyStats(weekStart, weekEnd)
            └─ SQL: SELECT strftime('%Y-%m-%d', end_time/1000, 'unixepoch'), SUM(actual_duration)
                     FROM focus_sessions
                     WHERE end_time >= ? AND end_time < ? AND is_completed = 1
                     GROUP BY date

  loadingCount 计数器：6 项加载全部完成后 → isLoading.postValue(false)
  图表数据自动补齐缺失日期（填 0）
```

### 2.5 主题实时同步全链路

```
用户在设置页点击切换"深色模式"
  │
  ▼
SettingsScreen.kt → SettingsViewModel.setThemeMode(ThemeMode.DARK)
  │
  ▼
SettingsViewModel.java:setThemeMode()
  └─ userPreferences.setThemeMode(THEME_MODE_DARK)
       │
       ▼
     UserPreferences.java (DataStore)
       └─ dataStore.edit { it[KEY_THEME_MODE] = THEME_MODE_DARK }
            │
            ▼
          DataStore 写入磁盘 → Flow 发射新值
            │
            ▼
          ComposeEntry.kt
            └─ collectAsState() 收到新值
                 │
                 ▼
               FocusFlowTheme(themeMode = DARK) 重组
                 │
                 ▼
               所有子组件自动跟随切换主题
```

### 2.6 任务列表响应式全链路

```
用户在输入框输入"写周报"并添加
  │
  ▼
TaskInputBar.kt → TaskListViewModel.addTask("写周报")
  │
  ▼
TaskListViewModel.java:addTask()
  └─ taskRepository.insertTask(new Task("写周报"))
       │
       ▼
     TaskRepository.java:insertTask()
       └─ taskDao.insertTask(task)
            │
            ▼
          TaskDao.java (Room 自动生成 SQL)
            └─ INSERT INTO tasks (title, is_completed, created_at) VALUES (?, 0, ?)
                 │
                 ▼
               focusflow.db 数据变更
                 │
                 ▼
             Room 自动推送 Flowable<List<Task>> 新数据
                 │
                 ▼
             TaskListViewModel 收到 → activeTasks.postValue(newList)
                 │
                 ▼
             TaskListScreen.kt 自动重组 (recomposition)
                 └─ LazyColumn 显示新任务
```
