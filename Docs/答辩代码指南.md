## 一、先搞清楚整体架构（一句话回答）

> **问：你这个项目的架构是什么？**
>
> 答：MVVM + Repository 模式。View（Compose UI）通过 ViewModel 获取数据，ViewModel 通过 Repository 操作 Room 数据库和 DataStore 配置。

```
UI (Compose) ↔ ViewModel ↔ Repository ↔ Room DB / DataStore
```

## 五、计时器功能（回答"计时是怎么实现的"，重点！）

### 5.1 三个状态

```java
// TimerViewModel.java:38-42
enum TimerState { IDLE, RUNNING, STOPPED }
```

- 白色环 = IDLE / STOPPED
- 绿色环 = RUNNING

### 5.2 计时核心逻辑

**开始计时（`startTimer()`）：**
1. 状态 → RUNNING，颜色 → 绿色
2. 启动 `Observable.interval(1, TimeUnit.SECONDS)` — 每秒 +1 秒
3. 创建 `FocusSession` 记录写入 Room（`focusRepository.startSession()`）
4. 启动前台服务 `TimerService`

**停止计时（`stopTimer()`）：**
1. 停止 `Observable` 定时器
2. 状态 → STOPPED，颜色 → 白色
3. 更新 FocusSession（写入 actualDuration、endTime）
4. 累加 Task.totalFocusSeconds
5. 停止前台服务

### 5.3 前后台协同

**前台（TimerViewModel）：** 管理计时状态、创建/更新数据库记录
**后台（TimerService）：** Foreground Service + WakeLock，保证不被系统杀死

两者**独立保存数据**，避免丢失：
- ViewModel 的 `stopTimer()` 保存一次
- Service 的 `stopTimerAndSave()` / `onDestroy()` 再保存一次（安全网）

### 5.4 环形进度条（`CircularTimer.kt`）

Canvas 绘制两层：
- 背景：半透明白色圆环（360°）
- 前景：彩色弧线，从顶部 -90° 开始

角度计算：`(elapsedSeconds % 3600) / 3600 * 360°`，60 分钟转一圈。

动画：`animateFloatAsState` 300ms 平滑过渡。

**关键文件：**
- `TimerViewModel.java:33-318` — 全部计时逻辑
  - `startTimer()`:136-174 — 开始计时
  - `stopTimer()`:179-232 — 停止计时
  - `toggleTimer()`:124-131 — 环形区域点击切换
  - `resumeSession()`:237-256 — 恢复未完成的计时
  - `startForegroundService()`:260-269 — 启动前台服务
- `TimerService.java:42-328` — 后台服务
  - `startTimer()`:94-116 — 启动 Observable 定时器 + 通知
  - `stopTimerAndSave()`:157-180 — 停止 + 保存

- `CircularTimer.kt:36-120` — 环形进度条绘制

## 九、面试/答辩常问问题 & 回答模板

### Q1: 为什么用 Java + Kotlin 混合？
> 数据层（Entity、DAO、ViewModel）是用 Java 写的，UI 层（Compose）用 Kotlin。两种语言在 Android 开发中完全兼容，可以在同一个项目中自由混用。Java 在数据层更稳妥，Kotlin 在 UI 层更简洁。

### Q2: 为什么用 RxJava 而不是协程？
> RxJava 提供 Observable.interval 来实现每秒触发的计时器，非常适合计时功能。配合 LiveData 将数据从 ViewModel 传给 UI。当然 Kotlin Flow + 协程也可以做到。

### Q3: 计时器退出后怎么恢复？
> 在 `TimerViewModel.init()` 中会调用 `focusRepository.getUnfinishedSession()` 检查数据库中是否有 `end_time IS NULL AND is_completed = 0` 的未完成记录。如果有且 taskId 匹配，就调用 `resumeSession()` 恢复计时。

### Q4: 同时点击多次"开始工作"会怎样？
> `toggleTimer()` 做了状态判断：IDLE/STOPPED → 开始，RUNNING → 停止。快速点击时状态已经切换，不会出现重复计时的问题。

### Q5: 为什么用 Foreground Service 而不是 WorkManager？
> 计时器需要精确到每秒更新，并且用户需要实时看到通知栏的倒计时。WorkManager 用于延迟/定期任务，不适用于这种秒级实时交互场景。Foreground Service 是 Android 官方推荐的实时后台任务方案。

### Q6: 统计查询为什么要用 end_time 而不是 start_time？
> 注释里写了（FocusSessionDao.java:52-53）：用 end_time 过滤，跨天的计时会话会被计入完成当天，与任务卡片上的累计时长保持一致。

### Q7: 连续打卡天数怎么算的？
> 从今天开始，每天往前检查是否有专注记录，一直查到没有记录的那天为止，中间不能断。
