# FocusFlow — 语音任务 + 番茄钟 PRD

## 1. 产品概述

### 1.1 产品定位
一款以任务为驱动核心的专注效率工具。用户通过键盘快速创建任务，一键开始专注计时，完成后查看时间投入统计。

### 1.2 核心流程
```
键盘创建任务 → 任务列表 → 点击"开始工作" → 计时 → 计时结束 → 回到任务列表
```

### 1.3 目标用户
- 需要快速记录灵感/待办的学生
- 希望减少操作摩擦的上班族
- 习惯"先列任务再执行"的效率爱好者

## 2. 功能模块

### 2.1 模块总览

| 模块 | 阶段 | 说明 |
|------|--------|------|
| 任务管理 | P0 | 键盘创建、列表展示、CRUD、关联计时 |
| 番茄钟 | P0 | 从任务进入计时，正向计时动画，后台运行 |
| 专注统计 | P1 | 日/周/月统计，图表展示，任务耗时分布 |
| 设置 | P1 | 主题切换、通知偏好 |


### 2.2 任务管理 (P0) — 首页

#### 2.2.1 创建任务
- **键盘创建**：点击输入框 → 打字输入 → 回车或点击发送按钮创建
- **附加信息**：键盘创建时可附加截止日期

#### 2.2.2 任务列表
- 默认按创建时间倒序排列
- 每张任务卡片展示：
  - 任务标题
  - 截止日期（如有，过期标红）
  - 累计专注时长（如 "已专注 2h30m"），不足1分钟显示秒（如 "45s"），超过1分钟只显示分钟
  - 右侧「开始工作」按钮
- 滑动删除：左滑出现删除按钮
- 勾选完成：复选框标记完成，已完成任务移至列表底部折叠区
- 直接开始工作：在列表卡片上点击「开始工作」按钮即可进入计时

### 2.3 计时功能 (P0) — 从任务进入

#### 2.3.1 计时模式
- **专注计时**：自由计时模式，用户自行决定何时停止
- **无预设时长**：不限制专注时长，完全由用户主动控制

#### 2.3.2 计时页面
- 顶部显示当前任务标题（面包屑导航，可返回任务列表）
- 居中环形进度条（画布动画）
- 环形内：已用时 MM:SS + “专注中”标签
- 底部单个控制按钮：
  - 初始状态：白色（未开始）
  - 点击后：变绿色（正在计时）
  - 再次点击：变白色（停止计时、记录数据、返回主页）
- 返回主页或退出应用时：自动停止计时（无论按钮状态）

#### 2.3.3 后台运行
- Foreground Service 保证不被系统杀死
- 通知栏显示：任务标题 + 已用时间 + 暂停按钮
- 应用被杀死后计时继续，重新打开自动恢复

#### 2.3.4 计时完成
- 用户要曲按下按钮（变成白色）止住正向计时
- 自动记录一条 FocusSession（关联当前任务，实际计时算已用时间）
- 计时停止后保持计时页，用户需手动点击返回按钮回到主页
- 返回主页或退出应用时也会停止计时
- 重新进入同一任务时，计时器恢复上次的累计专注时间（STOPPED 状态，可继续计时）

### 2.4 专注统计 (P1)

#### 2.4.1 统计维度
- **今日概览**：番茄钟数、总专注时长、完成任务数
- **本周**：每日柱状图，标注最高/最低日
- **本月**：趋势折线图
- **任务分布**：环形图展示各任务耗时占比
- **连续打卡**：连续专注天数

#### 2.4.2 页面布局
- 顶部大数字卡片（今日专注时长，核心指标）
- 时间维度切换标签（日/周/月）
- 柱状图区域
- 任务分布环形图

### 2.5 设置 (P1)

#### 2.5.1 外观
- 深色模式 / 浅色模式 / 跟随系统
- 主题色（预设 6 种颜色）

#### 2.5.2 通知
- 计时结束通知开关
- 提示音选择
- 震动开关

## 4. 技术架构

### 4.1 架构模式
MVVM + Repository Pattern

### 4.2 技术栈

| 类别 | 技术 |
|------|------|
| UI | Jetpack Compose + Material3 |
| 本地数据库 | Room |
| 配置存储 | DataStore Preferences |
| 导航 | Navigation Compose |
| 后台计时 | Foreground Service |
| 图表 | Vico |
| 异步 | RxJava3 + LiveData |

### 4.3 数据模型

#### Task
```java
@Entity(tableName = "tasks")
public class Task {
    @PrimaryKey(autoGenerate = true)
    private long id;

    @NonNull
    private String title;

    @NonNull
    private String description = "";

    @Nullable
    private Long dueDate;          // timestamp

    private boolean isCompleted = false;

    private long totalFocusSeconds = 0;  // 累计专注秒数，冗余字段方便排序

    private long createdAt = System.currentTimeMillis();

    private long updatedAt = System.currentTimeMillis();

    // getters and setters...
}
```

#### FocusSession
```java
@Entity(
    tableName = "focus_sessions",
    foreignKeys = @ForeignKey(
        entity = Task.class,
        parentColumns = "id",
        childColumns = "taskId",
        onDelete = ForeignKey.CASCADE
    )
)
public class FocusSession {
    @PrimaryKey(autoGenerate = true)
    private long id;

    private long taskId;            // 关联任务

    private int plannedDuration;    // 计划时长（秒）

    private int actualDuration;     // 实际完成时长（秒）

    private long startTime;

    @Nullable
    private Long endTime;

    @ColumnInfo(name = "is_completed")
    private boolean isCompleted = false;

    // getters and setters...
}
```

## 5. UI 设计要点

### 5.1 任务列表（首页）
- 顶部固定区域：输入框 + 发送按钮
- 任务卡片列表（列表）
- 每张卡片：标题 + 副标题 → 右侧「开始工作」按钮

  - 再点击变白色，停止计时，保持计时页
- 环内：已用时数字 + "专注中"标签
- 用户需手动点击返回箭头返回主页

### 5.3 统计页面
- 顶部大数字卡片：今日专注时长
- 时间维度标签切换
- 柱状图
- 环形图（任务占比）

## 7. 验收标准

### 7.1 任务管理

| 功能 | 验收标准 |
|------|---------|
| 键盘创建任务 | 输入文字 → 回车或点击发送按钮 → 列表刷新，新任务出现在列表顶部，输入框清空 |
| 空格/空输入校验 | 输入纯空格或空内容时，不允许创建任务，无任何反应 |
| 任务持久化 | 创建任务后，退出应用再重新打开，任务列表数据不丢失 |
| 任务编辑 | 点击任务标题 → 进入编辑模式 → 修改文字 → 确认保存 → 列表更新 |
| 任务删除 | 左滑任务卡片 → 点击删除按钮 → 弹出确认弹窗 → 确认删除 → 任务从列表移除，数据库删除 |
| 取消删除 | 删除确认弹窗中点击取消 → 弹窗关闭，任务保持不变 |
| 任务完成 | 点击复选框 → 任务标记为已完成 → 灰色文字 + 删除线 → 移至列表底部折叠区 |
| 取消完成 | 已折叠区中点击已完成任务的复选框 → 恢复为未完成 → 回到列表上方 |
| 截止日期 | 创建任务时可附加截止日期，过期任务截止日期显示为红色 |
| 累计专注时长 | 任务卡片正确显示累计专注时长（格式：Xh Ym），未专注过的任务不显示 |

### 7.2 计时功能

| 功能 | 验收标准 |
|------|---------|
| 进入计时页 | 点击任务卡片「开始工作」按钮 → 跳转计时页 → 顶部显示任务标题 + 返回箭头 |
| 初始状态 | 进入计时页时，环形进度条为白色，环内显示 "00:00" + "专注中"标签 |
| 开始计时 | 点击环形进度条 → 颜色变为绿色 → 环内时间开始正向计时（秒级更新） |
| 计时精度 | 计时误差不超过 1 秒（与系统时间对比） |
| 环形动画 | 进度条弧线从 0° 顺时针平滑增长至 360°，动画流畅不卡顿（60fps） |
| 停止计时 | 再次点击环形进度条 → 颜色变为白色 → 计时停止 → 环内时间定格 |
| 保持页面 | 停止计时后，保持在计时页，不自动跳转回主页 |
| 返回主页 | 点击返回箭头 → 返回任务列表页 → 自动停止计时并保存数据 |
| 退出应用 | 在计时过程中退出应用 → 自动停止计时并保存数据 |
| 快速点击 | 快速连续点击环形进度条，不会出现状态错乱或重复计时 |
| 横竖屏切换 | 计时中旋转屏幕，计时状态不丢失，时间继续正确显示 |

### 7.3 后台计时

| 功能 | 验收标准 |
|------|---------|
| 前台服务启动 | 开始计时后，Foreground Service 启动，通知栏显示计时通知 |
| 通知内容 | 通知栏显示：任务标题 + 已用时间（实时更新） |
| 切后台继续 | 按 Home 键切到后台 → 计时继续 → 通知栏时间实时更新 → 5 分钟后切回前台，时间正确 |
| 通知暂停 | 点击通知栏暂停按钮 → 计时暂停 → 通知更新为暂停状态 |
| 通知继续 | 点击通知栏继续按钮 → 计时恢复 → 通知更新为计时中状态 |
| 通知停止 | 点击通知栏停止按钮 → 计时停止 → 数据保存 → 通知移除 |
| 应用被杀死 | 系统杀死应用进程 → 重新打开应用 → 自动恢复计时状态，时间正确 |
| 电池优化 | 计时 30 分钟，电池消耗不超过 5%（待机状态下） |

### 7.4 数据关联

| 功能 | 验收标准 |
|------|---------|
| 计时记录创建 | 停止计时后，FocusSession 表新增一条记录，taskId 关联正确 |
| 累计时长更新 | 多次在同一任务上计时，Task.totalFocusSeconds 累加正确 |
| 实际时长 | FocusSession.actualDuration 等于实际计时秒数（误差 ≤ 1 秒） |
| 级联删除 | 删除任务时，关联的 FocusSession 记录同步删除 |
| 数据一致性 | 任务的 totalFocusSeconds 与该任务所有 FocusSession 的 actualDuration 之和一致 |
