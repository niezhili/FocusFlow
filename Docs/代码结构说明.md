### 数据层（data）

| 文件 | 行数 | 说明 |
|------|------|------|
| **AppDatabase.java** | 56 | **Room 数据库单例**。DCL 双重检查锁定。数据库名 `focusflow.db`，版本 1，包含 Task 和 FocusSession 两个实体。 |
| **Task.java** | 110 | **任务实体**（`tasks` 表）。字段：id、title、description、dueDate、isCompleted、totalFocusSeconds（反范式化冗余字段）、createdAt、updatedAt。 |
| **FocusSession.java** | 110 | **专注记录实体**（`focus_sessions` 表）。字段：id、taskId（外键→Task.id，CASCADE 级联删除）、plannedDuration（预留）、actualDuration、startTime、endTime、isCompleted。 |
| **TaskDao.java** | 71 | **任务 DAO 接口**。提供 `Flowable<List<Task>>` 响应式查询（getAllTasks、getActiveTasks、getCompletedTasks、searchTasks），CRUD 操作（insert、update、delete），完成状态切换，累计时长累加。 |

## 架构数据流图

```
用户操作 → Compose UI (Screen.kt)
              ↓ 观察 LiveData
           ViewModel (.java)
              ↓ 调用 Repository 方法
           Repository (.java)
              ↓ 调用 DAO 方法
           Room DAO (接口)
              ↓ Room 自动执行 SQL
           SQLite 数据库 (focusflow.db)
              ↑ Flowable 自动推送数据变化
           Repository
              ↑
           ViewModel (更新 LiveData)
              ↑
           Compose UI (自动重组)
              ↑
           用户看到更新后的界面
```

**后台计时路径（独立于 UI）：**

```
TimerService (Foreground Service)
  ↓ START Action
  ├── 持有 WakeLock（CPU 不休眠）
  ├── 通知栏常驻（任务标题 + 时间 + 按钮）
  ├── Observable.interval 每秒更新通知
  └── 停止时：独立保存 session 到 Room（安全网）
```

## 三态状态机（计时器核心）

```
       点击环形区域
  IDLE ────────────→ RUNNING
   ↑                  │
   │  点击环形区域     │ 点击环形区域
   │                  │
   └── STOPPED ←──────┘
        (自动保存)

颜色：IDLE=白  RUNNING=绿  STOPPED=白
```
